## 阶段七：互联网检索对照

| 来源 | 关键发现 | 应用情况 |
|------|----------|----------|
| FastAPI 生产部署最佳实践 (CSDN) | Gunicorn + Uvicorn workers 组合 | ⚠️ 建议：当前仅单进程 uvicorn，高并发下建议 `gunicorn -w 4 -k uvicorn.workers.UvicornWorker` |
| RAG 延迟优化 (CSDN, 51CTO) | 流式输出（SSE）、Prompt 压缩、TTL 缓存 | ⚠️ 建议：AI 问答可用 SSE 流式输出改善体验 |
| FAISS 并发安全 (CSDN) | `faiss.Index.add()` 非线程安全，需加锁 | ✅ 当前 `_faiss_insert` 使用 `_meta_lock`，但 FAISS 索引写入建议额外加锁 |
| GZip 中间件 (官方文档) | FastAPI 内置 `GZipMiddleware` 可减少 60-80% JSON 体积 | 🆕 已实施 |

## 风险与建议

### 低风险
1. **请求级超时**：当前 uvicorn 仅 `timeout_keep_alive=300`，无 `timeout_graceful_shutdown`
2. **速率限制缺失**：无 anti-DoS 保护

### 建议（后续）
1. **流式 AI 回答**：AI 问答改用 SSE/streaming，改善用户体感
2. **Gunicorn 多 worker**：生产部署建议 `gunicorn -w 4`
3. **FAISS 写入锁**：`_faiss_insert` 中 `_index.add()` 建议单独加锁
4. **搜索分页**：大量结果时支持分页加载
5. **慢查询日志**：> 1s 的查询单独记录
