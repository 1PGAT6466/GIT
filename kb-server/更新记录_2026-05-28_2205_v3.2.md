## 阶段二：后端代码扫描 (server.py)

### 🐛 Bug Fix（严重）

| 问题 | 位置 | 严重级别 | 状态 |
|------|------|----------|------|
| `_save_config()` 函数缩进崩塌 | server.py 第 2364-2391 行 | 🔴 Critical | ✅ 已修复并部署 |

**详情：** `_save_config()` 函数体中 `if os.path.exists(_CONFIG_PATH):` 及其后的 `try/except` 块完全没有正确缩进到函数内部，导致：
1. `_save_config()` 函数体实际为空（`os.makedirs` 之后的代码在模块级别）
2. 模块级别的 `if/try` 在服务启动时即触发会干扰模块加载
3. 虽然 Python 编译通过（语法上 `if` 在模块级合法），但运行时 `_save_config` 不保存任何配置
4. 所有管理面板的「工具管理」「FAQ 管理」「配置回滚」功能实际不持久化

**修复：** 将 `if`/`try`/`except` 全部加上正确的 4 空格缩进，`_save_config` 现在正常保存 config.json 并归档历史版本。

## 风险提示

1. **模型加载延迟**：服务重启后 Sentence-Transformer (BAAI/bge-base-zh-v1.5) 需要 3-5 秒加载，此期间 `/api/health` 等端点可能超时（健康探测建议设 10s 超时）
2. **单点故障**：当前为单机部署，无冗余；建议未来考虑 gunicorn + 多 worker 或容器化
3. **FAISS 索引单机限**制：全量索引在内存，大规模（>10 万条）可能需迁移至 Milvus/Weaviate

---

## 建议（非紧急）

1. 服务启动时返回 HTTP 503 + Retry-After 而不是连接超时
2. 添加 `/api/health/ready` 独立端点（K8s readiness probe）
3. 增加 Prometheus metrics 端点 `/api/metrics`
4. 考虑 SQLite 替代 metadata.json（大元数据时更高效）
5. 前端搜索结果支持键盘导航（↑↓ Enter）
6. 管理面板添加系统日志 viewer（journalctl 集成）
